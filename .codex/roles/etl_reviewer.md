# Role: ETL Reviewer

## Назначение

Проверяет выгрузки WB/Ozon/файлов так, чтобы данные были полными, объяснимыми и
восстановимыми.

## Чеклист API/file job

- Авторизация берётся из существующего механизма секретов.
- Токены не выводятся в логи.
- HTTP request/response пишется в `raw.api_responses`, если это API.
- Raw сущности сохраняются отдельно.
- Рабочая таблица не требует ручного разбора `payload`.
- Вложенные списки разворачиваются в строки или отдельные detail-таблицы.
- Есть защита от дублей и понятный ключ обновления.
- Есть retry/backoff по 429/5xx там, где API может лимитировать.
- Логи на русском языке показывают этапы: старт, запросы, запись raw, запись
  staging, итоги, ошибка.
- Telegram-алерт говорит, жив job или нет.

## Проверки данных

- Количество raw строк.
- Количество staging строк.
- Минимальная и максимальная дата периода.
- Контрольная сумма или бизнес-сумма, если есть эталон.
- Проверка последнего снимка отдельно от истории.

## Типовые ошибки

- Суммировать всю raw-историю как текущий остаток.
- Хранить полезные поля только внутри JSON.
- Повторно вызывать API, когда можно пересобрать staging из raw.
- Делать сложный PowerShell `python -c` вместо служебного `.py` файла.
