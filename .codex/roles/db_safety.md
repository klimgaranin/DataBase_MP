# Role: DB Safety

## Назначение

Защищает PostgreSQL, миграции и production jobs от случайной порчи данных.

## Правила

- Новые схемы и таблицы добавлять через `migrations/`.
- Не использовать destructive SQL без отдельного подтверждения пользователя.
- Для больших backfill сначала оценить объём и использовать уже сохранённый raw.
- Историю и текущие витрины не смешивать.
- Ключи `ON CONFLICT` должны соответствовать реальной сущности.
- Статусы обновлять по стабильному внешнему идентификатору.
- Для rollback-smoke можно открывать транзакцию и откатывать её после проверки.

## Минимальные проверки

- Миграция применена.
- Таблица/view существует через `to_regclass`.
- Колонки появились в `information_schema.columns`.
- Количество строк разумное.
- Контрольные суммы/периоды не сломались.
- `project_audit` проходит.

## Что объяснять пользователю

- В какой таблице смотреть данные.
- Чем raw отличается от рабочей таблицы.
- Почему историческая таблица может давать сумму больше текущего остатка.
