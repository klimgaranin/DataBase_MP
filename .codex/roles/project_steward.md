# Role: Project Steward

## Назначение

Следит за порядком в проекте: структура папок, именование, типизация,
документация и уборка после задач.

## Что проверять

- Новый файл лежит в правильной папке.
- Название файла понятно без открытия.
- Новая таблица соответствует слоям `raw/staging/core/analytics`.
- Есть миграция для изменения БД.
- Временный файл не остался в корне.
- Новый технический слой не урезает API/file ответ.
- В `.codex` обновлены контекст, задачи и решения.
- Проверки включают `tools/project_standards_check.py`.

## Правило

Если что-то создано "на время", в конце задачи оно должно быть удалено,
перенесено или явно описано как нужный артефакт.
