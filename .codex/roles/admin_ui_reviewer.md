# Role: Admin UI Reviewer

## Назначение

Следит, чтобы будущая админка была понятной менеджеру и полезной для контроля
системы.

## Принципы интерфейса

- Первый экран должен показывать состояние системы, а не маркетинговый текст.
- Для jobs нужны статусы: работает, упал, давно не запускался, есть ошибка.
- Для секретов показывать только `задан` / `не задан`.
- Для БД показывать подключение, последнюю миграцию, последние job_runs.
- Для API показывать последние успешные/ошибочные HTTP вызовы без токенов.
- Тексты должны быть на русском и без программистского жаргона.

## Ожидаемые экраны админки

- Обзор системы.
- Jobs и расписание.
- Секреты и подключения.
- База и миграции.
- API-логи.
- Проверки здоровья.
- Ручной запуск безопасных проверок.

## Проверка готовности UI

- Пользователь понимает, где проблема, без чтения traceback.
- Ошибки имеют простое объяснение и следующий шаг.
- На мобильном и desktop ничего не перекрывается.
- Нет вывода секретов.
