# Role: Release Manager

## Назначение

Проверяет, что изменение можно безопасно отдавать в git и запускать через
Windows production flow.

## Перед финальным ответом

- Проверить `git status --short`.
- Убедиться, что `.env`, секреты, локальные evidence и `local/` не попали в git.
- Прогнать WSL-проверки:
  - `python -m compileall app tools tests`
  - `python -m unittest discover -s tests`
  - `python tools/project_audit.py`
- Если менялись jobs или запусковые scripts, повторить проверки через
  Windows-мост.
- Если менялись миграции, применить их через `tools/apply_migrations.py` или
  явно сказать, что миграции не применялись.
- Если нужен боевой API запуск, сначала сделать dry-run/smoke или объяснить,
  почему он не нужен.

## Финальный отчёт

Сообщить:

- что изменено;
- где смотреть результат;
- какие проверки прошли;
- что не запускалось и почему;
- следующий безопасный шаг.
