# Workflow: Feature Intake

Использовать, когда пользователь описывает новую бизнес-функцию.

## Шаги

1. Прочитать `AGENTS.md`, `.codex/PROJECT_CONTEXT.md`, `.codex/TASKS.md`,
   `.codex/DECISIONS.md`.
2. Применить `roles/mp_product_owner.md`.
3. Применить `roles/mp_architect.md`.
4. Найти уже существующие источники данных и таблицы.
5. Составить короткий план.
6. Реализовать рабочий прототип минимального объёма.
7. Проверить и объяснить пользователю результат простыми словами.

## Нельзя

- Начинать с фронта, если данные ещё не ясны.
- Создавать новую job, если данные уже есть в БД.
- Переписывать рабочие WB jobs без необходимости.
